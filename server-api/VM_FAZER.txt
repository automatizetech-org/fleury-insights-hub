# O que fazer na VM — passo a passo

## 1. Colar a pasta server-api

Coloque a pasta `server-api` na VM (ex.: `C:\Users\ROBO\Documents\server-api`).

---

## 2. Criar o .env

Na pasta `server-api`, copie o exemplo para `.env`:

```powershell
cd C:\Users\ROBO\Documents\server-api
copy .env.example .env
```

(O .env.example já vem preenchido. Só conferir se BASE_PATH está certo: `C:\Users\ROBO\Documents`.)

---

## 3. Parar o que está na 3001

Se o backend do WhatsApp estiver rodando na 3001:

```powershell
netstat -ano | findstr :3001
```

Anote o PID (última coluna) e mate o processo:

```powershell
taskkill /PID <número_do_pid> /F
```

---

## 4. Subir o backend do WhatsApp na porta 3010

Inicie o backend do WhatsApp como você já faz, mas na porta **3010** (variável de ambiente ou config, conforme o projeto dele). Exemplo se ele usar variável PORT:

```powershell
set PORT=3010
cd C:\Users\ROBO\...\backend\whatsapp-emissor
node launcher.js
```

Deixe esse terminal aberto. O WhatsApp fica só na 3010 (interno na VM).

---

## 5. Subir o server-api na 3001

Em **outro** terminal:

```powershell
cd C:\Users\ROBO\Documents\server-api
npm install
npm start
```

Deixe aberto. Deve aparecer: `API unificada em http://localhost:3001` e `Proxy WhatsApp: http://localhost:3010`.

---

## 6. Ngrok

O ngrok continua apontando para **3001** (não muda nada):

```powershell
ngrok http --domain=plagiaristic-elinore-ungloomily.ngrok-free.dev 3001
```

---

## Resumo

| O quê              | Porta | Onde        |
|--------------------|-------|-------------|
| server-api         | 3001  | VM (ngrok)  |
| Backend WhatsApp   | 3010  | VM (só interno) |

O site continua usando a mesma URL do ngrok para tudo (arquivos + WhatsApp).
